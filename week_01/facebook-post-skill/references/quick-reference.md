# Facebook 智慧發文 - 快速參考指南

## 常用提示詞樣板

### 場景 1：直接發佈完整文本

**用戶說法**：
```
幫我在 Facebook 上發佈這篇文章：
[粘貼完整文本]
```

**Claude 回應流程**：
1. 確認文本無誤
2. 打開 Facebook
3. 自動發佈
4. 顯示成功結果

### 場景 2：主題寫文章再發佈

**用戶說法**：
```
幫我寫一篇關於 [主題] 的 Facebook 文章，包括 [具體要求]
```

**範例**：
```
幫我寫一篇介紹 OpenCLI 的 Facebook 文章，
要包括：
- 工具的核心功能
- 支援的平台
- 實際應用場景
- 最後簽名為「Claude Code 發的文」
```

**Claude 回應流程**：
1. 根據主題撰寫文章
2. 向用戶展示內容
3. 確認後發佈
4. 顯示結果

### 場景 3：編輯後再發佈

**用戶說法**：
```
幫我改一下這篇文章，然後發佈到 Facebook：
[提供原始文本]

改進點：
- [要求 1]
- [要求 2]
```

**Claude 回應流程**：
1. 根據要求編輯文本
2. 向用戶展示修改後的版本
3. 確認後發佈

### 場景 4：排程多篇文章

**用戶說法**：
```
幫我發佈這些文章到 Facebook（依次發佈）：

文章 1：[内容]
文章 2：[内容]
文章 3：[内容]
```

**Claude 回應流程**：
1. 確認所有文章
2. 逐篇發佈
3. 顯示發佈結果

## 故障排除速查表

### 問題 → 解決方案

**Q: Chrome 擴展無法連接**
A: 
1. 確保 Chrome 瀏覽器已打開
2. 確保 Claude in Chrome 擴展已安裝
3. 檢查擴展是否已啟用（chrome://extensions）
4. 切換回 Claude 標籤頁面
5. 重新授予權限

**Q: 找不到發文框**
A:
1. 確認 Facebook 頁面已完全加載
2. 嘗試重新整理頁面
3. 使用替代查詢詞（"發文框" 或 "post input"）
4. 檢查是否在正確的 Facebook 頁面（不是設定頁面等）

**Q: 文字無法輸入**
A:
1. 確認發文對話框已完全打開
2. 重新點擊文字框以確保焦點
3. 等待 1 秒後再重試
4. 檢查是否有彈窗或通知覆蓋了輸入框

**Q: 無法發佈**
A:
1. 檢查文本框內容是否完整
2. 確認「發佈」按鈕可見
3. 等待 2 秒再截圖確認
4. 如果仍未成功，手動檢查是否有驗證錯誤

**Q: 發佈超時**
A:
1. 等待額外 3-5 秒
2. 手動刷新 Facebook 檢查文章是否已發佈
3. 如果已發佈，告訴用戶成功
4. 如果未發佈，嘗試重新發佈

## 批量發佈管理

### 多篇文章發佈流程

**對於 N 篇文章**：

```
步驟 1：收集所有文章
- 獲取所有文章內容
- 驗證每篇文章的格式和內容

步驟 2：展示並確認
「我準備依次發佈以下 N 篇文章：
1. [標題 1]
2. [標題 2]
...
確定嗎？」

步驟 3：依次發佈
For i in range(1, N+1):
    - 發佈第 i 篇
    - 等待 2 秒確認
    - 顯示結果
    - 如果是最後一篇，等待 2 秒讓頁面穩定

步驟 4：總結
「全部發佈完成！

發佈統計：
✅ 成功：N 篇
❌ 失敗：0 篇

你現在可以在 Facebook 上看到所有的文章。」
```

## 性能和最佳實踐

### 推薦做法

✅ **DO：**
- 等待頁面完全加載再操作
- 在重要操作前後拍攝截圖
- 為長文本或複雜操作提供進度更新
- 在用戶確認後再執行破壞性操作
- 提供清晰的錯誤消息和恢復選項

❌ **DON'T：**
- 快速連續執行點擊而不等待
- 跳過確認步驟
- 在未驗證的情況下進行操作
- 忽視錯誤和異常狀況
- 進行未授權的操作
